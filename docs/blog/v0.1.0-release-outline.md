## 3. Getting Started (60-second setup)

### Install the package

```shell
dotnet add package Fritz.Aspire.Hosting.Minecraft --version 0.1.0
```

### Wire it upin your AppHost

```csharp
using Aspire.Hosting.Minecraft;

var builder = DistributedApplication.CreateBuilder(args);

var redis = builder.AddRedis("cache");
var api = builder.AddProject<Projects.MyApi>("api");

// One line to add a Minecraft server
var minecraft = builder.AddMinecraftServer("minecraft", gamePort: 25565, rconPort: 25575)
    .WithBlueMap(port: 8100)           // 3D web map in the Aspire dashboard
    .WithOpenTelemetry()               // JVM metrics â†’ Aspire dashboard
    .WithAspireWorldDisplay<Projects.Aspire_Hosting_Minecraft_Worker>()
    .WithMonitoredResource(api)        // Each resource gets its own in-world cube
    .WithMonitoredResource(redis);

builder.Build().Run();
```

### Run and connect

```shell
cd samples/MinecraftAspireDemo/MinecraftAspireDemo.AppHost
dotnet run
```

Open Minecraft Java Edition â†’ Add Server â†’ `localhost:25565` â†’ join and look around.

## 4. Feature Highlights

### 4a. Minecraft Server as a First-Class Aspire Resource

- `AddMinecraftServer()` spins up a [Paper](https://papermc.io/) server in Docker via the trusted `itzg/minecraft-server` image.
- RCON health check baked in â€” other services can `.WaitFor()` the Minecraft server just like any database or cache.
- The server appears in the Aspire dashboard with endpoints, logs, and health status.

```csharp
var minecraft = builder.AddMinecraftServer("minecraft");
// That's it. You have a Minecraft server.
```

> **Screenshot placeholder:** Aspire dashboard showing the Minecraft resource with game, RCON, and BlueMap endpoints.

### 4b. OpenTelemetry â€” JVM Meets .NET

- `.WithOpenTelemetry()` bind-mounts the OpenTelemetry Java agent into the container.
- JVM metrics (heap, GC, threads, CPU) flow into the same Aspire dashboard as your .NET services.
- Game metrics (TPS, MSPT, player count) are polled via RCON and exported as custom OTEL metrics.

| Metric | What It Tells You |
|---|---|
| `jvm.memory.used` | Is the server running out of heap? |
| `minecraft.tps` | Is the server keeping up? (target: 20.0) |
| `minecraft.mspt` | How long each tick takes |
| `minecraft.players.online` | Who's connected |
| `minecraft.rcon.latency_ms` | RCON round-trip health |

> **Screenshot placeholder:** Aspire dashboard metrics page showing JVM + Minecraft metrics side-by-side.

### 4c. BlueMap â€” A 3D Web Map in Your Dashboard

- `.WithBlueMap()` installs the BlueMap plugin and exposes its web UI as an HTTP endpoint.
- Click the endpoint in the Aspire dashboard and you get a full 3D, interactive map of your Minecraft world â€” including the Aspire resource structures.

> **Screenshot placeholder:** BlueMap web UI showing the in-world Aspire structures from above.

### 4d. In-World Health Display

- `.WithAspireWorldDisplay<TWorker>()` registers a background worker that connects to the Minecraft server via RCON.
- Each `.WithMonitoredResource()` gets:
  - **A torch-topped structure** â€” emerald blocks for healthy, redstone blocks for unhealthy.
  - **A floating hologram** â€” resource name and current status, updated in real time.
  - **A scoreboard entry** â€” sidebar scoreboard showing all resources at a glance.
- Player chat messages announce health changes as they happen.

> **Screenshot placeholder:** In-world view showing multiple resource cubes with holograms and the scoreboard sidebar.
